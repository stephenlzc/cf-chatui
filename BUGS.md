## 🔴 紧急 (Critical) - 影响核心功能

### BUG-001: GPT-OSS-120B 模型响应问题
- **状态**: 待修复
- **描述**: GPT-OSS-120B 模型在某些情况下不返回响应或响应为空
- **重现步骤**: 
  1. 选择 GPT-OSS-120B 模型
  2. 发送任意消息
  3. 观察无响应或响应卡住
- **影响**: 用户无法使用 GPT-OSS-120B 模型
- **可能原因**: 
  - 该模型可能需要特定的消息格式
  - 流式响应处理不兼容
  - 模型本身响应时间较长导致超时
- **临时解决**: 使用 GLM-4.7-Flash 模型替代

### BUG-002: 前端 Token 过期检查缺失
- **状态**: 待修复
- **描述**: 本地存储的 JWT token 过期后，前端不会自动跳转到登录页
- **重现步骤**:
  1. 登录并等待 24 小时（token 过期时间）
  2. 发送消息
  3. 收到 401 错误，但界面仍显示已登录
- **影响**: 用户体验差，需要手动刷新页面
- **建议修复**: 
  - 在发送请求前检查 token 过期时间
  - 收到 401 时自动清除 token 并跳转登录

### BUG-003: SSE 流式响应中断问题
- **状态**: 待修复
- **描述**: 在某些浏览器或网络环境下，SSE 流式响应可能意外中断
- **重现步骤**:
  1. 使用移动网络或不稳定网络
  2. 发起长对话请求
  3. 响应可能在传输中途停止
- **影响**: 对话不完整，需要重新发送
- **建议修复**: 添加重连机制和超时检测

## 🟠 高优先级 (High) - 影响用户体验

### BUG-004: 中文编码问题
- **状态**: 待修复
- **描述**: 某些中文字符在响应中显示为乱码或方框
- **重现步骤**:
  1. 发送包含特殊中文标点的消息
  2. 观察响应中的编码问题
- **影响**: 中文用户体验不佳
- **可能原因**: TextDecoder 编码设置不正确
- **临时解决**: 避免使用特殊字符

### BUG-005: 聊天记录刷新丢失
- **状态**: 待修复
- **描述**: 页面刷新后，当前对话记录完全丢失
- **重现步骤**:
  1. 进行多轮对话
  2. 刷新浏览器页面
  3. 对话历史清空
- **影响**: 无法延续之前的对话上下文
- **建议修复**: 
  - 使用 localStorage/IndexedDB 持久化聊天记录
  - 添加导出/导入对话功能

### BUG-006: 无请求超时处理
- **状态**: 待修复
- **描述**: AI 请求没有设置超时，可能无限等待
- **重现步骤**:
  1. 在弱网环境下发送请求
  2. 请求卡住无响应
  3. 无法取消或重试
- **影响**: 用户界面假死，体验差
- **建议修复**: 
  - 添加 AbortController 实现请求取消
  - 设置合理的超时时间（如 30 秒）
  - 添加取消按钮

### BUG-007: 错误信息不够友好
- **状态**: 待修复
- **描述**: 错误提示过于技术化，用户难以理解
- **重现步骤**:
  1. 断开网络连接
  2. 发送消息
  3. 显示 "请求失败: 500" 等错误
- **影响**: 普通用户不知道如何处理
- **建议修复**: 
  - 将技术错误转换为用户友好的提示
  - 添加重试按钮和操作指引

## 🟡 中优先级 (Medium) - 功能优化

### BUG-008: 图片生成无实时预览
- **状态**: 待修复
- **描述**: 调整图片生成参数（如 steps）时无实时预览效果
- **影响**: 用户难以预估参数对结果的影响
- **建议修复**: 添加参数说明和示例图片

### BUG-009: 缺少加载状态指示
- **状态**: 待修复
- **描述**: 某些操作（如模型切换、嵌入生成）缺少加载指示
- **影响**: 用户不确定操作是否在进行
- **建议修复**: 统一添加加载动画和进度提示

### BUG-010: 系统提示词硬编码
- **状态**: 待修复
- **描述**: 系统提示词写死在代码中，用户无法自定义 AI 行为
- **影响**: 无法根据需求调整 AI 角色和风格
- **建议修复**: 
  - 添加系统提示词配置界面
  - 支持保存多个预设角色

### BUG-011: 无输入长度限制
- **状态**: 待修复
- **描述**: 输入框没有字符数限制，可能发送超长内容
- **影响**: 
  - 可能导致 API 错误或高额费用
  - 前端性能问题
- **建议修复**: 
  - 添加字符数限制（如 4000 字符）
  - 显示当前字符数和限制

### BUG-012: GLM 模型响应延迟
- **状态**: 已知，非代码问题
- **描述**: GLM-4.7-Flash 模型使用非流式模式，响应有延迟
- **影响**: 用户体验不如流式响应流畅
- **说明**: 这是为了修复思考过程显示问题的权衡方案
